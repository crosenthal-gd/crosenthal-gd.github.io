(constant) fs

changes webpack settings for modules and bluesky. adds/modifies/removes plugins following changes are made to webpack config - fixBabelLoader - the web sdk packages are installed in node_modules which are ignored by default by bable transpiler. This will add just the web sdk packages to be included from node_modueles - addDecoratorsLegacy - adds decorators support which is mainly used by mobx https://github.com/arackaf/customize-cra/blob/master/api.md#adddecoratorslegacy https://babeljs.io/docs/babel-plugin-proposal-decorators - disableEsLint - disables the default eslint rules from react scripts since websdk has its own rules - addNodePolyfill - some polyfills for using node functions in browser(may not be used anymore) - fixPublicPath - the default public path set by react scritps is '/'. we use that for dev, but since files are included from cdn domains in prod, we reset it to auto https://webpack.js.org/guides/public-path/#automatic-publicpath - addModuleFederationModulePlugin - this is only done for modules. Adds the module federation plugin in production mode to use shared packages from bluesky - fixMiniCSSExtractPlugin - insert css into body rather than head - addModuleFederationAppPlugin - only added for bluesky app. sets up module federation plugin. exposes all shared packages that will be used by modules - addBlueSkyInlineScriptPlugin - Adds a webpack plugin to change the index script in BlueSky App This plugin receives the static resources to be included in index.html and loads the same script from cdn domain - addBundleVisualizer - adds bundle visuyalizer if EUREKA_BUNDLE_VISUALIZE env var is true. This can be used to debug bundle size
